<a id="sec-1-introduction"></a>
## 1. 简介

本指南记录了 Top Speed 当前使用的自定义赛道格式，内容基于当前解析器与运行时代码路径整理而成。它是一份创作参考文档，而不是宣传概览，因此核心目标是实用且准确。

一个自定义赛道文件会同时控制四个方面：道路形状与路面流向、启用的天气模型及其过渡行为、车辆穿过各赛道段时所应用的声学房间配置，以及放置在赛道上的可选自定义声源。这些系统都通过 id 相互关联，并且会在赛道被接受前完成验证。

这意味着，制作赛道不仅仅是写出合法的键。你还需要理解：哪些键会真正被运行时消费，哪些键只是元数据，极端值经过钳制后会如何变化，以及哪些引用必须事先存在。

<a id="sec-2-how-to-imagine-a-track-shape-as-an-author"></a>
## 2. 作为作者，该如何想象赛道形状

最准确的心智模型，是把整条赛道看作一条会循环重放的赛道段时间线。赛道段顺序，就是赛道顺序；赛道段长度，是以米为单位的真实物理距离；弯道类型则决定玩家在通过该赛道段时，车道中心会以多快的速度向左或向右偏移。

`straight` 会让横向中心偏移保持为零。左右弯道则会以不同强度产生连续偏移。由于偏移是在整段长度上累计计算的，因此在弯道类型相同的情况下，长赛道段会比短赛道段产生更大的累计中心位移。

当前运行时使用的弯道强度系数如下：

- `straight`: `0`
- `easy_left` / `easy_right`: `0.5`
- `left` / `right`: `0.6667`
- `hard_left` / `hard_right`: `1.0`
- `hairpin_left` / `hairpin_right`: `1.5`

赛道段宽度与弯道形状是分开的。如果 `width > 0`，该赛道段就会使用它自己的宽度；如果 `width <= 0`，运行时会回退到默认车道宽度。这会显著影响赛道手感，因为完全相同的弯道序列，在不同宽度配置下，可能会显得宽容，也可能会显得苛刻。

一个实用的工作流是：先只编写赛道段顺序、长度和宽度，先开起来、调整节奏；等几何形状手感正确之后，再加入天气、房间和自定义声音。

<a id="sec-3-folder-layout-file-discovery-and-loading-rules"></a>
## 3. 文件夹布局、文件发现与加载规则

自定义加载既可以接受文件夹路径，也可以接受文件路径。如果传入的是文件夹，加载器会扫描该文件夹中的 `*.tsm` 文件，并按照不区分大小写的排序顺序选取文件名排在最前的那个；如果传入的是文件路径，它必须是一个位于现有文件夹中的 `.tsm` 文件。

推荐布局如下：

```text
Tracks/
  MyTrack/
    track.tsm
    Audio/
      ambience_loop.ogg
      birds_1.ogg
```

声源部分中的声音路径，是相对于 `tsm` 文件所在文件夹来解析的。它们不会相对于游戏根目录解析，也不会相对于 `Sounds` 资源根目录解析。这样可以保证每个自定义赛道都是自包含的。

自定义赛道的显示名称解析顺序如下：

1. 如果 `meta.name` 已定义且非空，则优先使用它。
2. 使用父文件夹名称。
3. 使用不带扩展名的文件名。

<a id="sec-4-tsm-file-format-rules"></a>
## 4. TSM 文件格式规则

TSM 是基于行的格式，并且只支持两种语句形式：部分头和键值赋值。

部分头格式：

```text
[kind]
[kind:id]
```

键值格式：

```text
key = value
```

行内注释以 `#` 开始，并会在解析前被剥离。空行会被忽略。

支持的部分类型包括 `meta`、`segment`、`weather`、`room` 和 `sound`。其中 `segment`、`weather`、`room` 与 `sound` 都要求带 id（即 `[kind:id]`）；`meta` 不使用 id（即 `[meta]`）。

未知的部分类型会导致验证失败。严格部分中的未知键同样会导致验证失败，唯一例外是那些以 `meta` 或 `metadata` 开头的扩展式键。

部分顺序是灵活的。解析器会自上而下读取，并在遇到新的部分头时，将前一个部分对象封存完成。同一部分中重复同一个键是允许的；在该部分对象内部，后写的值会覆盖先前的值。

键名和部分类型在解析时会被规范化为小写，并把空格与连字符转换成下划线。id 会先去掉首尾空白，并以不区分大小写的方式匹配。id 唯一性是按部分家族（`segment`、`weather`、`room`、`sound`）分别强制执行的，因此同一家族内出现重复 id 会被视为验证错误。

<a id="sec-5-sections-and-how-they-connect"></a>
## 5. 各部分以及它们如何关联

赛道至少需要一个 `segment` 部分。此外，`meta.weather` 也是必填项，并且它必须指向一个已存在的天气配置 id。如果这两者任何一个缺失，赛道都会被拒绝。

赛道段是整条赛道的骨架，它们可以引用：

- 天气配置 id
- room id 或内置房间预设名称
- 声源 id

声源还可以通过 `start_area` 和 `end_area` 来引用赛道段 id，因此赛道段 id 同时会被几何系统和音频触发逻辑使用。

房间行为有两层覆盖关系：

- room 部分中的值会覆盖房间预设的基线值
- segment 中的房间覆盖键，只会覆盖该赛道段所选中的房间

天气行为也有两层：

- 来自 `meta.weather` 的赛道级默认配置
- 可选的赛道段级天气 id，以及可选的混合时间（`weather_transition_seconds`）

<a id="sec-6-detailed-key-reference"></a>
## 6. 详细键参考

<a id="sec-6-1-meta-section"></a>
## 6.1 Meta 部分

本部分的头写法是 `[meta]`。

`name`  
这是自定义赛道在菜单和播报中使用的显示名称。  
允许的值：
- 任意非空文本值。

`version`  
这是可选的作者元数据，用于你自己的版本修订。  
它会和赛道数据一起存储，但本身不会直接改变运行时的驾驶或音频行为。  
允许的值：
- 任意非空文本值。

`weather`  
这是赛道必填的默认天气配置 id。  
允许的值：
- 任意已存在的天气部分 id。
- 例如：如果文件中包含 `[weather:clear]`，那么 `weather = clear` 就是合法的。
- 如果该 id 不存在，加载会失败。

`ambience`  
这个键用于选择旧式环境循环音家族。  
允许的值：
- `none`  
  禁用旧式环境循环音播放。
- `desert`  
  使用沙漠环境循环音行为。
- `airport`  
  使用机场环境循环音行为。

别名值：
- `0`: `none`
- `noambience`: `none`
- `1`: `desert`
- `2`: `airport`

<a id="sec-6-2-weather-section"></a>
## 6.2 天气配置部分

部分头写法为 `[weather:<id>]`，例如 `[weather:clear]`。

`kind`  
这个键用于选择该配置的基础预设。  
当设置了这个键时，运行时会先加载该预设值，然后再应用同一个天气部分中显式写出的数值键。  
允许的值：
- `sunny`  
  晴天基线，风力中性、能见度较长。
- `rain`  
  雨天配置基线，更强调湿润环境音。
- `wind`  
  大风配置基线，具有更强的横风。
- `storm`  
  暴风雨配置基线，天气强度最高。

别名值：
- `0`: `sunny`
- `1`: `rain`
- `rainy`: `rain`
- `2`: `wind`
- `windy`: `wind`
- `3`: `storm`
- `stormy`: `storm`

`longitudinal_wind_mps`  
这是沿驾驶轴方向的风分量，单位是米/秒。  
正值相当于迎风，会提高相对风速；负值相当于顺风。  
允许的值：
- 任意有限数值。

`lateral_wind_mps`  
这是横风分量，单位是米/秒。  
它会把侧向空气速度计入空气阻力大小。  
允许的值：
- 任意有限数值。

`air_density`  
它控制空气动力计算中使用的空气密度，单位为 kg/m³。  
允许的值：
- 任意大于 `0` 的数值。

运行时说明：如果构造阶段仍然收到无效的非正值，就会回退为 `1.225`。

`drafting_factor`  
这是阻力模型中直接作用于空气阻力的乘数。  
`1.0` 表示中性；大于 `1.0` 会增加阻力；小于 `1.0` 会减少阻力。  
允许的值：
- 任意大于 `0` 的数值。

运行时说明：构造阶段的钳制会强制它不低于 `0.1`。

`temperature_c`  
它以摄氏度存储天气温度，用于配置状态。  
当前驾驶阻力逻辑并不直接依赖这个键，但它仍然是天气配置数据的一部分。  
允许的值：
- 任意有限数值。

`humidity`  
它以归一化比值形式存储相对湿度。  
允许的值：
- 任意位于 `0` 到 `1` 之间的数值。

运行时说明：在天气配置构建时，值会被钳制到 `0..1`。

`pressure_kpa`  
它以 kPa 为单位存储大气压。  
允许的值：
- 任意大于 `0` 的数值。

运行时说明：如果构造阶段收到无效的非正值，会回退为 `101.325`。

`visibility_m`  
它以米为单位存储天气配置的能见度距离。  
允许的值：
- 任意大于 `0` 的数值。

运行时说明：如果构造阶段收到无效的非正值，会回退为 `20000`。

`rain_gain`  
这个值设定该配置中雨声循环的天气强度增益。  
允许的值：
- 任意大于或等于 `0` 的数值。

运行时说明：配置内部会在钳制后存储 `0..4`；当前播放路径在真正应用到声源音量前，还会再做一次最终 `0..1` 钳制。

`wind_gain`  
这个值设定该配置中风声循环的天气强度增益。  
允许的值：
- 任意大于或等于 `0` 的数值。

运行时说明：配置内部会在钳制后存储 `0..4`；当前播放路径在真正应用到声源音量前，还会再做一次最终 `0..1` 钳制。

`storm_gain`  
这个值设定该配置中暴风声循环的天气强度增益。  
允许的值：
- 任意大于或等于 `0` 的数值。

运行时说明：配置内部会在钳制后存储 `0..4`；当前播放路径在真正应用到声源音量前，还会再做一次最终 `0..1` 钳制。

天气切换行为：  
当某个赛道段设置了 `weather = some_id` 时，运行时会逐步混合过渡到该配置。  
混合速度由赛道段键 `weather_transition_seconds` 控制。

<a id="sec-6-3-room-section"></a>
## 6.3 Room 部分

部分头写法为 `[room:<id>]`，例如 `[room:tunnel_a]`。

`name`  
这是该房间配置的可选可读标签。  
允许的值：
- 任意非空文本值。

`room_preset`  
这个键用于选择初始化房间声学时所使用的内置预设基线。  
同一部分中的任何数值型房间键，都会覆盖这条基线。  
允许的值：
- 第 7.2 节列出的任意预设 id。

`reverb_time`  
它控制房间混响尾音的总衰减时长，单位为秒。  
允许的值：
- 任意数值。
- 运行时会将其钳制为非负。

`reverb_gain`  
它控制房间混响回传的整体电平。  
允许的值：
- 任意数值。
- 运行时会钳制到 `0..1`。

`hf_decay_ratio`  
它控制高频在混响尾音中相对低频/中频的衰减速度。  
值越低，高频越快变暗；值越高，高频亮度保留得越久。  
允许的值：
- 任意数值。
- 运行时会钳制到 `0..1`。

`late_reverb_gain`  
它控制后期尾音成分（后期反射）的增益。  
允许的值：
- 任意数值。
- 运行时会钳制到 `0..1`。

`diffusion`  
它控制反射的密度和平滑度。  
值越低，反射听起来越分散、颗粒感越重；值越高，反射会更顺滑、更稠密。  
允许的值：
- 任意数值。
- 运行时会钳制到 `0..1`。

`air_absorption`  
它控制声音在空气中传播时损失多少高频内容。  
值越低，远处声源仍会保持较亮；值越高，远处声音会更快变闷。  
允许的值：
- 任意数值。
- 运行时会钳制到 `0..1`。

`occlusion_scale`  
它控制障碍物对直达声路径的削弱程度。  
值越低，障碍物的遮挡感越弱；值越高，障碍物的遮挡感越强。  
允许的值：
- 任意数值。
- 运行时会钳制到 `0..1`。

`transmission_scale`  
它控制声音穿过墙体或重型结构等边界的泄漏程度。  
值越低，隔墙声音越弱、阻隔感越强；值越高，穿透过去的声音越多。  
允许的值：
- 任意数值。
- 运行时会钳制到 `0..1`。

`occlusion_override`  
这是一个可选的直接遮挡值。  
一旦设置，它会覆盖房间/配置路径中派生出来的遮挡量。  
允许的值：
- 任意数值。
- 设置后，运行时会钳制到 `0..1`。

`transmission_override`  
它设置一个传输覆盖值，并把它同时应用到低、中、高三个频段。  
当你想要一个全局传输覆盖，而不是逐频段微调时，就可以用它。  
允许的值：
- 任意数值。
- 设置后，运行时会钳制到 `0..1`。

`transmission_override_low`  
它设置低频段传输覆盖值。  
允许的值：
- 任意数值。
- 设置后，运行时会钳制到 `0..1`。

`transmission_override_mid`  
它设置中频段传输覆盖值。  
允许的值：
- 任意数值。
- 设置后，运行时会钳制到 `0..1`。

`transmission_override_high`  
它设置高频段传输覆盖值。  
允许的值：
- 任意数值。
- 设置后，运行时会钳制到 `0..1`。

`air_absorption_override`  
它设置一个空气吸收覆盖值，并同时作用于低、中、高三个频段。  
允许的值：
- 任意数值。
- 设置后，运行时会钳制到 `0..1`。

`air_absorption_override_low`  
它设置低频段空气吸收覆盖值。  
允许的值：
- 任意数值。
- 设置后，运行时会钳制到 `0..1`。

`air_absorption_override_mid`  
它设置中频段空气吸收覆盖值。  
允许的值：
- 任意数值。
- 设置后，运行时会钳制到 `0..1`。

`air_absorption_override_high`  
它设置高频段空气吸收覆盖值。  
允许的值：
- 任意数值。
- 设置后，运行时会钳制到 `0..1`。

赛道段引用行为：  
一个赛道段可以引用 room 部分 id，也可以直接引用预设 id。  
如果赛道段级房间覆盖存在，那么它们会在选中的房间基线之后再应用。

<a id="sec-6-4-segment-section"></a>
## 6.4 Segment 部分

部分头写法为 `[segment:<id>]`，例如 `[segment:s12]`。

`type`  
这个键用于设置该赛道段的弯道类别。  
所选类别会影响道路中心在这段距离推进过程中如何偏移。  
允许的值：
- `straight`  
  无横向偏移。
- `easy_left`  
  轻微向左偏移。
- `left`  
  中等向左偏移。
- `hard_left`  
  强烈向左偏移。
- `hairpin_left`  
  非常强的向左偏移。
- `easy_right`  
  轻微向右偏移。
- `right`  
  中等向右偏移。
- `hard_right`  
  强烈向右偏移。
- `hairpin_right`  
  非常强的向右偏移。

别名值：
- `0`: `straight`
- `1`: `easy_left`
- `2`: `left`
- `3`: `hard_left`
- `4`: `hairpin_left`
- `5`: `easy_right`
- `6`: `right`
- `7`: `hard_right`
- `8`: `hairpin_right`

`surface`  
这个键用于设置赛道段的路面类别。  
允许的值：
- `asphalt`  
  标准铺装路面行为。
- `gravel`  
  松散路面行为。
- `water`  
  湿滑/低抓地行为。
- `sand`  
  柔软路面行为。
- `snow`  
  类雪地低抓地行为。

别名值：
- `0`: `asphalt`
- `1`: `gravel`
- `2`: `water`
- `3`: `sand`
- `4`: `snow`

`noise`  
这个键用于设置旧式赛道段噪音行为。  
允许的值：
- `none`  
  无旧式赛道段噪音。
- `crowd`  
  人群环境音行为。
- `ocean`  
  海洋环境音行为。
- `runway`  
  跑道/飞机行为。
- `clock`  
  时钟环境音行为。
- `jet`  
  喷气机事件行为。
- `thunder`  
  雷声事件行为。
- `pile`  
  堆场环境音行为。
- `construction`  
  施工环境音行为。
- `river`  
  河流环境音行为。
- `helicopter`  
  直升机事件行为。
- `owl`  
  猫头鹰事件行为。

<a id="sec-6-6-value-parsing-and-token-rules"></a>
## 6.6 值解析与标记规则

数值解析使用 invariant-culture，因此小数点必须写成 `.`。  
枚举解析不区分大小写，并且容忍标记文本中出现空格、连字符和下划线。  
布尔值解析接受 `1/0`、`true/false`、`yes/no` 和 `on/off`。

向量键必须严格写成三个由逗号分隔的浮点值（`x,y,z`）。  
在严格部分中，未知键会被视为验证错误，除非键名以 `meta` 或 `metadata` 开头。  
声路径安全规则会拒绝根路径、带盘符路径，以及路径穿越片段（即 `.` 和 `..` 这样的路径组件）。  
交叉引用验证会在解析结束后执行，用来确保被引用的天气 id、房间 id、赛道段 id 和声源 id 全部存在。

<a id="sec-7-1-reverb-parameter-meanings"></a>
## 7.1 混响参数含义

本节会用通俗语言解释每一个房间参数。

| 参数 | 它控制什么 | 增大后会发生什么 |
|---|---|---|
| `reverb_time_seconds` | 混响尾音在完全衰减前会持续多久。 | 空间听起来会更长、更有残留感。 |
| `reverb_gain` | 混响回传的整体强度。 | 相较于干声，反射声会更响。 |
| `hf_decay_ratio` | 高频在尾音中相对其他频段能保留多久。 | 高频内容会更久地保持明亮。 |
| `late_reverb_gain` | 后期、稠密尾音成分的电平。 | 后期混响主体会更强、更明显。 |
| `diffusion` | 早期与后期反射的稠密度/平滑度。 | 反射会显得更不离散、更融合、更平顺。 |
| `air_absorption` | 声音在空气中随距离造成的高频损失。 | 远处声音会损失更多亮度与清晰度。 |
| `occlusion_scale` | 障碍物削弱直达声的强度。 | 障碍物的遮挡效果会更强。 |
| `transmission_scale` | 声音穿过墙体/结构的程度。 | 你能听到更多穿透边界泄漏出来的声音。 |

<a id="sec-9-validation-and-troubleshooting"></a>
## 9. 验证与故障排查

在构建运行时对象之前，赛道加载会先经过一个严格的验证阶段。错误会直接阻止加载；警告则允许继续加载，但某些值可能会被钳制规则调整。

常见的硬失败以及直接修复方法：

| 失败项 | 为什么会发生 | 修复方法 |
|---|---|---|
| 缺少默认天气配置 | `meta.weather` 为空，或指向了未知 id | 在 `meta` 中定义 `weather = some_id`，并添加 `[weather:some_id]` 部分 |
| 部分 id 重复 | 同一家族中的两个部分复用了同一个 id | 重命名 id，确保每个 `segment`、`weather`、`room` 和 `sound` id 都唯一 |
| 未知枚举标记 | 值文本无法映射到受支持的枚举标记/整数 | 使用本指南中的合法标记，或合法的数字枚举值 |
| 严格部分中出现未知键 | 键名拼写错误，或该键根本不受支持 | 修正拼写；自定义标签请放在 `meta*` / `metadata*` 前缀下 |
| 未知引用 id | 赛道段或声音引用了不存在的 id | 添加缺失的目标部分，或修正引用 id 文本 |
| 无效的声路径 | 使用了绝对路径、带盘符路径或路径穿越 | 只使用赛道相对路径，例如 `Audio/wind.ogg` |
| 无效向量 | 位置值不是严格的 `x,y,z` 浮点格式 | 严格写成由逗号分隔的三个浮点值 |

需要特别注意的非致命行为：

- 小于 50 的赛道段长度会给出警告，并被钳制到 50。
- 许多房间和天气增益字段会在运行时构造器中被钳制。
- 对于声源，如果 `pitch <= 0`，会变成 `1.0`。
- `width <= 0` 不会导致失败；运行时会回退到默认车道宽度。

<a id="sec-10-practical-authoring-workflow"></a>
## 10. 实用创作流程

一个稳定的工作流，是把制作分成四轮，并且每轮都测试，而不是一口气把所有系统一次性写完。

第一轮只做几何：`meta`、一个天气配置，以及包含 `type`、`length`、`surface`、`width` 的赛道段。先确认节奏、弯道流动感和圈速手感。

第二轮做天气过渡：加入额外的天气配置，然后只在你确实想要发生天气切换的地方，为赛道段添加 `weather` 和 `weather_transition_seconds`。确认过渡是否平滑，并且是否发生在预期的赛道段边界上。

第三轮做声学：为隧道、封闭结构等特定区域加入房间预设或房间部分。如果需要局部微调，请优先使用赛道段级房间覆盖，而不是复制大量房间部分。

第四轮做声源：先从一个路径已知正确的简单声源开始，然后再逐步加入空间化行为、区域触发和随机变体。验证来自赛道段的每一个声源 id 引用，以及来自声触发器的每一个区域 id 引用。

最好再另外保留一份纯文本顺序说明，用来描述赛道段顺序、距离和设计意图。当行为不对时，把这份顺序说明与运行时表现一对照，就能更容易判断问题是出在几何、天气 id、房间选择，还是声触发引用上。
